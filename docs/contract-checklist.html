MVP Build & Marketing Contract Checklist
Scope • Ownership • Milestones • Security • Support
Purpose This checklist defines the baseline requirements for any build/marketing partner working on FundGuard Africa’s MVP and go-to-market. It protects code ownership, sets clear milestones, and establishes security expectations.

1) Scope of Work

Product (MVP)

  • Web app (clarify: desktop/mobile responsive) — frontend + backend + database.
  • Fraud simulation: live alerts, recovery flow, evidence report generation.
  • Auth & roles (basic/premium/institutional).
  • Deployment to www.fundguardafrica.com and GitHub Pages (investor demo).
  • Technical & user documentation.

Marketing (if included)

  • Channels: Google/Meta/LinkedIn (specify).
  • Ad spend is separate from build fee (state budget & billing).
  • Deliverables: # of graphics, videos, captions, landing pages.
  • Run period, targeting (regions, industries), reports cadence.

2) Ownership & Access

  • All source code, designs, content, and accounts belong to FundGuard Africa Ltd.
  • Code must be committed to FundGuard’s GitHub repos (not contractor’s private repos).
  • FundGuard retains admin access to hosting, analytics, domains, ad accounts.

3) Payments (Milestone-based)

  • 30% — Kickoff (project plan agreed & repo set up).
  • 40% — MVP deployed on FundGuard domain with agreed features working.
  • 30% — Final handover (docs, code, access, bug-fix window active).
  • No full payment until all deliverables are confirmed.

4) Timeline & Delivery

  • MVP Build: __ weeks
  • Deployment & polish: __ days
  • Marketing run (if any): __ weeks/months
  • Weekly demos; status reports every Friday.

5) Quality & Security Baseline

  • HTTPS enforced; secure auth; secrets not hard-coded.
  • Clean, documented code; handover includes setup instructions.
  • Data handling aligned to NDPR/GDPR posture (no sensitive PII stored unnecessarily).
  • Error monitoring + basic analytics enabled.

6) Post-Launch Support

  • Bug-fix window: __ days included after handover.
  • Optional paid support (SLA, hourly/day rate) — define in addendum.

7) Exit, Non-Lock-In & Refund

  • FundGuard can terminate for non-delivery; partial refund for undelivered milestones.
  • No vendor lock-in: full code + assets + credentials returned at termination.
